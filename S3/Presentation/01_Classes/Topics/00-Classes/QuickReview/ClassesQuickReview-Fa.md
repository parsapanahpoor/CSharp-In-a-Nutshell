# Classes - بررسی سریع

## تعریف
کلاس یک الگوی طراحی (Blueprint) است که برای ایجاد اشیاء با ویژگی‌ها و رفتارهای مشابه استفاده می‌شود.

## کاربرد
- سازماندهی کد و ایجاد ساختار منطقی
- کپسول‌کردن داده‌ها و رفتارها در یک واحد
- پایه برای ارث‌بری و چندریختی
- مدل‌سازی موجودیت‌های دنیای واقعی

## نکات کلیدی
- **Reference Type** هستند (در Heap ذخیره می‌شوند)
- می‌توانند **Constructor** داشته باشند (Default Constructor توسط کامپایلر ایجاد می‌شود)
- می‌توانند **Abstract** یا **Sealed** باشند
- پشتیبانی از **Inheritance** (تک‌وراثتی)
- پشتیبانی از **Interface** (چندگانه)
- می‌توانند **Generic** باشند
- می‌توانند **Nested Class** داشته باشند
- **Access Modifiers** را پشتیبانی می‌کنند (public, internal, private, protected, ...)
- می‌توانند **Partial** باشند (تقسیم در چند فایل)
