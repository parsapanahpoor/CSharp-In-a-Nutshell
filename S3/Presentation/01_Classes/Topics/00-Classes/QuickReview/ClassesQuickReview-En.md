# Classes - Quick Review

## Definition
A class is a blueprint (template) used to create objects with similar properties and behaviors.

## Usage
- Code organization and creating logical structure
- Encapsulating data and behaviors in a single unit
- Foundation for inheritance and polymorphism
- Modeling real-world entities

## Key Points
- **Reference Type** (stored in Heap)
- Can have **Constructors** (Default Constructor is generated by compiler)
- Can be **Abstract** or **Sealed**
- Support **Inheritance** (single inheritance)
- Support **Interfaces** (multiple interfaces)
- Can be **Generic**
- Can contain **Nested Classes**
- Support **Access Modifiers** (public, internal, private, protected, ...)
- Can be **Partial** (split across multiple files)
