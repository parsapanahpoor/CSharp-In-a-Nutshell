## ❓ Q12:

## protected internal چیه؟ با protected فرقش چیه؟

### ✅ جواب:

<div dir="ltr" style="text-align: left;">

```csharp
// Assembly A:
public class Base {
protected int A;              // فقط Derived Classes
internal int B;               // فقط همین Assembly
protected internal int C;     // Derived Classes یا همین Assembly ✅
private protected int D;      // Derived Classes در همین Assembly ✅
}

// Assembly B:
public class Derived : Base {
void Test() {
A = 1;  // ✅ OK (protected)
B = 2;  // ❌ Error (internal)
C = 3;  // ✅ OK (protected internal)
D = 4;  // ❌ Error (private protected)
}
}

// Assembly A (همان assembly):
public class SameAssemblyClass {
void Test() {
var obj = new Base();
obj.A = 1;  // ❌ Error
obj.B = 2;  // ✅ OK (internal)
obj.C = 3;  // ✅ OK (protected internal)
obj.D = 4;  // ❌ Error (not derived)
}
}

```

### 📊 جدول مقایسه:

| Access Modifier | همین کلاس | Derived (همین Assembly) | Derived (Assembly دیگر) | Same Assembly (Non-Derived) |
|----------------|-----------|------------------------|------------------------|----------------------------|
| **protected** | ✅ | ✅ | ✅ | ❌ |
| **internal** | ✅ | ✅ | ❌ | ✅ |
| **protected internal** | ✅ | ✅ | ✅ | ✅ |
| **private protected** | ✅ | ✅ | ❌ | ❌ |

<div dir="rtl" style="text-align: right;">

### 🎯 نکات کلیدی:

- **`protected internal`** = `protected` **OR** `internal` (گسترده‌تر)
- **`private protected`** = `protected` **AND** `internal` (محدودتر)
- **کاربرد:** برای کتابخانه‌ها که می‌خواهند Extension Points امندن
