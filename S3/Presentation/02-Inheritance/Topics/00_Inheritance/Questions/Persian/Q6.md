## ❓ Q6:

## Virtual Method Table (VMT) چیه؟ چطور کار می‌کنه؟

### ✅ جواب:

<div dir="ltr" style="text-align: left;">


```csharp
class Animal {
public virtual void Speak() => Console.WriteLine("Animal sound");
}

class Dog : Animal {
public override void Speak() => Console.WriteLine("Woof!");
}

Animal a = new Dog();
a.Speak(); // چطوری می‌فهمه باید Dog.Speak() رو صدا کنه؟

```

<div dir="rtl" style="text-align: right;">


**پشت صحنه:**

<div dir="ltr" style="text-align: left;">

```

[Dog Object in Memory]
└─ Type Handle → [Dog's VMT]
├─ ToString()     → Object.ToString
├─ GetHashCode()  → Object.GetHashCode
├─ Speak()        → Dog.Speak ✅ (Override شده)
└─ …

```

<div dir="rtl" style="text-align: right;">

### 🎯 مکانیزم:

1. Runtime متوجه می‌شه a در واقع یک Dog است
2. از Type Handle به VMT می‌رود
3. Speak() را از VMT پیدا می‌کند
4. Dog.Speak() را اجرا می‌کند ✅
