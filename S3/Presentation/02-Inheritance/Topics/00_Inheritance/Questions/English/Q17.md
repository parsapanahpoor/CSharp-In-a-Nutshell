## ‚ùì Q17:

## Why does C# only have Single Inheritance and why is Multiple Inheritance not allowed?

### ‚úÖ Deep Answer:

This is an **intentional design** to avoid **Diamond Problem** issues and Type System complexities.

### ‚ùå Diamond Problem:

<div dir="ltr" style="text-align: left;">

```csharp
// Suppose Multiple Inheritance was allowed:
class A {
    public void Method() => Console.WriteLine("A");
}

class B : A {
    public override void Method() => Console.WriteLine("B");
}

class C : A {
    public override void Method() => Console.WriteLine("C");
}

class D : B, C {  // ‚ùå Which Method() should it use?
    // D inherits from B and C
    // B.Method() or C.Method()?
}

D d = new D();
d.Method();  // ‚ùå Ambiguous! Which one?
```

<div dir="ltr" style="text-align: left;">

### üîç Problems with Multiple Inheritance:

**1. Ambiguity:**
- If two base classes have the same method, which one is used?

**2. Constructor Order:**
- What is the constructor call order?
- `B()` first or `C()` first?

**3. Virtual Method Resolution:**
- If both `B` and `C` have overridden `Method()`, which one executes?

**4. Memory Layout:**
- How are `B` and `C` fields laid out in memory?
- What if they have the same field name?

<div dir="ltr" style="text-align: left;">

```csharp
class B {
    public int X = 10;
}

class C {
    public int X = 20;  // ‚ùå Same name!
}

class D : B, C {  // Which is D.X? 10 or 20?
}
```

<div dir="ltr" style="text-align: left;">

### ‚úÖ C# Solution: Interfaces

<div dir="ltr" style="text-align: left;">

```csharp
// Instead of Multiple Inheritance:
interface IReadable {
    void Read();
}

interface IWritable {
    void Write();
}

class File : IReadable, IWritable {  // ‚úÖ Multiple Interface Implementation
    public void Read() { }
    public void Write() { }
}
```

<div dir="ltr" style="text-align: left;">

### üéØ Advantages of Single Inheritance:

1. **Simplicity**: Simpler and more understandable Type System
2. **Performance**: Faster resolution (no ambiguity)
3. **Type Safety**: Fewer errors at compile-time
4. **Maintainability**: Code is easier to maintain

### üí° Comparison with C++:

C++ has Multiple Inheritance but has many problems:
- Virtual Inheritance to solve Diamond Problem
- Complexity in Constructor/Destructor order
- Ambiguity resolution with `::` operator

C# with **Single Inheritance + Multiple Interfaces** doesn't have these problems!
