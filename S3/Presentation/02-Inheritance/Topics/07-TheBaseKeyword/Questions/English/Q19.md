## ‚ùì Q19:

## How does base work in Constructor Initialization Order?

### ‚úÖ Deep Answer:

`base` in Constructor is called **before Field Initialization**. Order is very important!

### üîç Initialization Order:

<div dir="ltr" style="text-align: left;">

```csharp
class Base {
    protected int baseField = Initialize("Base Field");
    
    public Base() {
        Console.WriteLine("Base Constructor");
    }
    
    private static int Initialize(string name) {
        Console.WriteLine($"Initializing: {name}");
        return 0;
    }
}

class Derived : Base {
    private int derivedField = Initialize("Derived Field");
    
    public Derived() : base() {  // ‚úÖ base is called first
        Console.WriteLine("Derived Constructor");
    }
    
    private static int Initialize(string name) {
        Console.WriteLine($"Initializing: {name}");
        return 0;
    }
}

new Derived();
```

<div dir="ltr" style="text-align: left;">

### üìä Exact Order:

1. **Base Static Fields** (if they exist)
2. **Base Static Constructor**
3. **Derived Static Fields**
4. **Derived Static Constructor**
5. **Base Instance Fields**
6. **Base Constructor** (from `base()`)
7. **Derived Instance Fields**
8. **Derived Constructor Body**

<div dir="ltr" style="text-align: left;">

### üéØ Advanced Note:

`base()` is called **before Derived Field Initialization**! This can cause bugs if we use Derived Fields in Base Constructor.

### ‚ö†Ô∏è Problem:

<div dir="ltr" style="text-align: left;">

```csharp
class Base {
    public Base() {
        Initialize();  // ‚ö†Ô∏è Virtual method in constructor
    }
    
    protected virtual void Initialize() { }
}

class Derived : Base {
    private int value = 10;  // Not yet initialized!
    
    protected override void Initialize() {
        Console.WriteLine(value);  // 0! ‚ùå (not yet initialized!)
    }
}
```

<div dir="ltr" style="text-align: left;">

### üí° Solution:

**Don't** use Virtual Methods in Constructor! Or use Initialize Pattern.
