## How do Virtual Methods work in Constructor?

### ‚ö†Ô∏è Problem:

Calling virtual method in constructor is **dangerous**! Because derived class is not yet initialized.

<div dir="ltr" style="text-align: left;">

```csharp
class Base {
    public Base() {
        Initialize();  // ‚ö†Ô∏è virtual method in constructor
    }
    
    protected virtual void Initialize() {
        Console.WriteLine("Base initialized");
    }
}

class Derived : Base {
    private int value = 10;  // Not yet initialized!
    
    protected override void Initialize() {
        Console.WriteLine($"Derived initialized: {value}");
        // value = 0! ‚ùå (not yet initialized)
    }
}

new Derived();
// Output: "Derived initialized: 0" ‚ùå
```

<div dir="ltr" style="text-align: left;">

### ‚úÖ Solution:

**Don't** use virtual methods in constructor! Or use Initialize pattern.

<div dir="ltr" style="text-align: left;">

```csharp
class Base {
    public Base() {
        // Don't call virtual method!
    }
    
    public void Initialize() {  // After construction
        DoInitialize();
    }
    
    protected virtual void DoInitialize() {
        Console.WriteLine("Base initialized");
    }
}

var obj = new Derived();
obj.Initialize();  // ‚úÖ Now all fields are initialized
```
