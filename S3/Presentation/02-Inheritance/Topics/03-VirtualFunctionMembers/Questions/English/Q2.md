## Why do we need Virtual Methods?

### ‚úÖ Answer:

Without virtual, we cannot change base class behavior in derived class.

<div dir="ltr" style="text-align: left;">

```csharp
// ‚ùå Without virtual:
class Animal {
    public void Speak() {  // Not virtual
        Console.WriteLine("Animal sound");
    }
}

class Dog : Animal {
    public void Speak() {  // This is Hiding, not Override!
        Console.WriteLine("Woof!");
    }
}

Animal animal = new Dog();
animal.Speak();  // Output: "Animal sound" ‚ùå (not "Woof!")
```

<div dir="ltr" style="text-align: left;">

### ‚úÖ With virtual:

<div dir="ltr" style="text-align: left;">

```csharp
class Animal {
    public virtual void Speak() {  // ‚úÖ virtual
        Console.WriteLine("Animal sound");
    }
}

class Dog : Animal {
    public override void Speak() {  // ‚úÖ override
        Console.WriteLine("Woof!");
    }
}

Animal animal = new Dog();
animal.Speak();  // Output: "Woof!" ‚úÖ
```

<div dir="ltr" style="text-align: left;">

### üéØ Note:

Virtual methods allow us to define **common behavior** and **change** it in derived classes.
