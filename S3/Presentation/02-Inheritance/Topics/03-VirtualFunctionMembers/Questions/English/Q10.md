## What is the difference between Virtual Methods and Interface Methods?

### ‚úÖ Comparison:

| Feature | Virtual Methods | Interface Methods |
|-------|----------------|-------------------|
| **Implementation** | Can have default | Must be implemented |
| **Multiple** | Only one base class | Multiple interfaces |
| **Access Modifier** | Can be protected | Always public |
| **Versioning** | Can be backward compatible | Breaking change |

<div dir="ltr" style="text-align: left;">

```csharp
// Virtual Method:
class Animal {
    public virtual void Speak() {
        Console.WriteLine("Animal sound");  // ‚úÖ Default implementation
    }
}

class Dog : Animal {
    public override void Speak() {
        Console.WriteLine("Woof!");
    }
}

// Interface:
interface IAnimal {
    void Speak();  // ‚ùå Must be implemented
}

class Dog : IAnimal {
    public void Speak() {  // ‚úÖ Must implement
        Console.WriteLine("Woof!");
    }
}
```

### üéØ Note:

Virtual Methods for **Inheritance Hierarchy**, Interface Methods for **Contracts**.
