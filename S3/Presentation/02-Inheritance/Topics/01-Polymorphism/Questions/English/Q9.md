## Can we not override a virtual method?

### ‚úÖ Answer:

Yes! It's not mandatory. If we don't override, the base class method executes.

<div dir="ltr" style="text-align: left;">

```csharp
class Animal {
    public virtual void Speak() {
        Console.WriteLine("Animal makes sound");
    }
}

class Dog : Animal {
    // Didn't override! ‚úÖ No problem
}

class Cat : Animal {
    public override void Speak() {
        Console.WriteLine("Meow!");
    }
}

Animal dog = new Dog();
dog.Speak();  // Output: "Animal makes sound" (from base class)

Animal cat = new Cat();
cat.Speak();  // Output: "Meow!" (from derived class)
```

<div dir="ltr" style="text-align: left;">

### üéØ Note:

Override is **optional**. If we don't do it, the base class behavior is used.
