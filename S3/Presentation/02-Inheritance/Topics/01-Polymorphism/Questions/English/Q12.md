## How does Overloading work with optional parameters?

### âœ… Answer:

Optional parameters can cause ambiguity in Overloading!

<div dir="ltr" style="text-align: left;">

```csharp
class Calculator {
    public int Add(int a, int b) => a + b;
    
    public int Add(int a, int b = 0, int c = 0) => a + b + c;
}

var calc = new Calculator();
calc.Add(2, 3);  // âŒ Error: Ambiguous call!
```

<div dir="ltr" style="text-align: left;">

### Error:
```
Error: The call is ambiguous between 'Add(int, int)' and 'Add(int, int, int)'
```

### âœ… Solution:

<div dir="ltr" style="text-align: left;">

```csharp
class Calculator {
    // Only one method with optional parameters:
    public int Add(int a, int b = 0, int c = 0) => a + b + c;
}

calc.Add(2);      // âœ… a=2, b=0, c=0
calc.Add(2, 3);   // âœ… a=2, b=3, c=0
calc.Add(2, 3, 4); // âœ… a=2, b=3, c=4
```
