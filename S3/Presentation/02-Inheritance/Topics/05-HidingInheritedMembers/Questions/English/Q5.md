## Why is Hiding dangerous?

### ‚ö†Ô∏è Problem:

Hiding can cause **confusion** because it doesn't have Polymorphism and creates unexpected behavior.

<div dir="ltr" style="text-align: left;">

```csharp
class Base {
    public void Process() {
        Console.WriteLine("Base Process");
    }
}

class Derived : Base {
    public new void Process() {
        Console.WriteLine("Derived Process");
    }
}

Base obj = new Derived();
obj.Process();  // Output: "Base Process" ‚ùå (not "Derived Process"!)

// This is unexpected behavior!
```

<div dir="ltr" style="text-align: left;">

### ‚úÖ Solution:

Use **virtual/override**:

<div dir="ltr" style="text-align: left;">

```csharp
class Base {
    public virtual void Process() {
        Console.WriteLine("Base Process");
    }
}

class Derived : Base {
    public override void Process() {
        Console.WriteLine("Derived Process");
    }
}

Base obj = new Derived();
obj.Process();  // Output: "Derived Process" ‚úÖ
```

<div dir="ltr" style="text-align: left;">

### üéØ Note:

**Use virtual/override as much as possible**! Hiding is only needed in special cases.
