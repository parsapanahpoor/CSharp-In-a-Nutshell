## What is the difference between Boxing/Unboxing and Casting?

### ‚úÖ Deep Answer:

**Boxing/Unboxing** is conversion between value type and object, but **Casting** is conversion between reference types.

### üîç Boxing vs Casting:

<div dir="ltr" style="text-align: left;">

```csharp
// Boxing: value type ‚Üí object
int number = 42;
object obj = number;  // ‚úÖ Boxing (value type to reference type)

// Unboxing: object ‚Üí value type
int num = (int)obj;   // ‚úÖ Unboxing (reference type to value type)

// Casting: reference type ‚Üí reference type
Animal animal = new Dog();
Dog dog = (Dog)animal;  // ‚úÖ Casting (reference to reference)
```

<div dir="ltr" style="text-align: left;">

### üî¨ Key Differences:

| Feature | Boxing/Unboxing | Casting |
|-------|----------------|---------|
| **Conversion Type** | Value ‚Üî Reference | Reference ‚Üî Reference |
| **Memory** | Copies (Boxing) | Only reference changes |
| **Performance** | Slower (heap allocation) | Fast (only type check) |
| **Exception** | InvalidCastException | InvalidCastException |

### ‚ö†Ô∏è Boxing Problem:

<div dir="ltr" style="text-align: left;">

```csharp
int x = 42;
object obj = x;  // Boxing: a new object is created in heap
int y = (int)obj;  // Unboxing: reads from heap

// Performance overhead:
// 1. Heap allocation (Boxing)
// 2. Memory copy (Boxing)
// 3. Type check (Unboxing)
// 4. Memory read (Unboxing)
```

<div dir="ltr" style="text-align: left;">

### ‚úÖ Casting (without Boxing):

<div dir="ltr" style="text-align: left;">

```csharp
Dog dog = new Dog();
Animal animal = dog;  // Only reference changes (no copy)
Dog dog2 = (Dog)animal;  // Only type check (very fast)
```

<div dir="ltr" style="text-align: left;">

### üéØ Advanced Note:

**Boxing** creates an **object wrapper** in heap that holds the value type. This is **costly**!
**Casting** only checks **type information** and has no memory allocation.

### ‚ö†Ô∏è Important:

**Avoid** Boxing as much as possible! It has performance overhead.
