## Why do we sometimes need to cast?

### ‚úÖ Answer:

When the compiler cannot **automatically** determine the type, we must **explicitly** cast.

<div dir="ltr" style="text-align: left;">

```csharp
class Animal { }
class Dog : Animal { }

// Example 1: When variable type is not clear
object obj = new Dog();
Dog dog = (Dog)obj;  // ‚úÖ Must cast

// Example 2: When using interface
IEnumerable<int> numbers = new List<int> { 1, 2, 3 };
List<int> list = (List<int>)numbers;  // ‚úÖ To access List methods

// Example 3: When using base reference
Animal animal = new Dog();
Dog dog2 = (Dog)animal;  // ‚úÖ To access Dog methods
```

<div dir="ltr" style="text-align: left;">

### üéØ General Rule:

- **Upcast**: Automatic (doesn't need cast)
- **Downcast**: Explicit (must cast)
- **Interface ‚Üí Concrete**: Explicit (must cast)

### ‚ö†Ô∏è Note:

If you need to cast a lot, maybe your **Design** has a problem. Better to use Polymorphism!
