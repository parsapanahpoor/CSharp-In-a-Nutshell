## When to use Sealed?

### âœ… Answer:

When we want to guarantee **Security**, **Performance** or **Design Integrity**.

### Examples:

1. **Utility Classes**: Helper classes that shouldn't be extended
2. **Value Types Wrappers**: Like System.String
3. **Security Classes**: Security classes
4. **Performance-Critical**: Classes where Performance is important

<div dir="ltr" style="text-align: left;">

```csharp
// Example: Utility Class
sealed class MathHelper {
    public static int Add(int a, int b) => a + b;
    // Don't want anyone to inherit from it
}
```
