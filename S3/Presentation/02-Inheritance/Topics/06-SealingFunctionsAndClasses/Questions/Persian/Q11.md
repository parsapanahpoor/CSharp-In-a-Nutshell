## ❓ Q11:

## چه موقع از Sealed استفاده کنیم؟

### ✅ جواب:

وقتی می‌خوایم **امنیت**، **Performance** یا **Design Integrity** رو تضمین کنیم.

### مثال‌ها:

1. **Utility Classes**: کلاس‌های کمکی که نباید extend بشن
2. **Value Types Wrappers**: مثل System.String
3. **Security Classes**: کلاس‌های امنیتی
4. **Performance-Critical**: کلاس‌هایی که Performance مهمه

<div dir="ltr" style="text-align: left;">

```csharp
// مثال: Utility Class
sealed class MathHelper {
    public static int Add(int a, int b) => a + b;
    // نمی‌خوایم کسی ازش ارث‌بری کنه
}
```

<div dir="rtl" style="text-align: right;">

### 🎯 نکته:

از Sealed استفاده کن وقتی می‌خوای **Extension** رو محدود کنی.
